Carousel

Carousel component is a component to build responsive carousels. It contains a collection of Images, like all other components exposed by this library and many other config objects. It displays images as a carousel, so a big image (either full or limited width) with right and left arrows and it shows images every a fixed amount of time. Optionally, it can display previews above/below the main image. Obviously, there are additional configurable features, like aria, keyboard navigation, sizes and so on. A very cool feature, it the integration with ModalGallery. In fact, you can configure the carousel to open the current image in a modal way, using ModalGalleryComponent.

If you need to support also Internet Explorer 11 (IE11), enable legacyIE11Mode: true and use only ABSOLUTE image paths (for instance '/assets/a.png' instead of './assets/a.png'). However, try to understand that I cannot support that obsolete browser forever!

Inputs

id
Type: number
A mandatory unique id >= 0 of the current carousel. Despite CarouselComponent doesn't use this value, it must be unique because a carousel can open a modal image using GalleryService.
images
Type: Image[]

The mandatory array of Image objects, that represent both the thumb gallery and the modal images. In fact, an Image is an object with both thumb and image source paths with other additional configuration parameters. For more info check Image.

Carousel uses the image path assigned to the modal property in Image class. This is for historical reason and to provide a better integration with modal-gallery.

Please, don't use this.images.push(img) when you want to modify this array, instead re-assign it to trigger change detection. For instance: this.images = [...this.images, newImage] to create a new array from the original with a new image at the end.

If you need IE11 support, always use ABSOLUTE image paths (for instance '/assets/a.png' instead of './assets/a.png').
carouselConfig
Type: CarouselConfigoptional

Interface to configure the carousel, like to change sizes, hide some features and so on.

For more information, please see the CarouselConfig's section
playConfig
Type: PlayConfigoptional

Interface to change the behaviour of auto-navigation, like autoPlay and interval in milliseconds.

For more information, please see the PlayConfig's section
carouselImageConfig
Type: CarouselImageConfigoptional

Interface to change current image of the carousel, like description or to invert the swipe direction.

For more information, please see the CarouselImageConfig's section
dotsConfig
Type: DotsConfigoptional

Interface to configure dots.

For more information, please see the DotsConfig's section
previewConfig
Type: CarouselPreviewConfigoptional

Interface to configure previews, for instance to choose either the number, visibility, arrows, sizes. If you want to know how to use this object, please check PreviewsComponent.
infinite
Type: booleanoptional

Input to enables infinite sliding. true by default
accessibilityConfig
Type: AccessibilityConfigoptional

Interface to customize accessibility features, like alt, titles, aria-labels and so on. CarouselComponent applies a default value with english texts. This default config object is exported as KS_DEFAULT_ACCESSIBILITY_CONFIG.
disableSsrWorkaround
Type: booleanoptional

Input to support SystemJS bypassing some workarounds to enable SSR support. false by default

Attention: use this only if you are using SystemJS instead of angular-cli or webpack. In future releases I'll drop SystemJS support, so please consider to switch to angular-cli as soon as possible.

Outputs

show

An ImageEvent right after the current image is changed.

It contains an Action enum and a result (boolean or number).

Everytime the current image changes, a new ImageEvent will be emitted.

  • action: either Action.NORMAL, Action.CLICK, Action.KEYBOARD or Action.SWIPE
  • result: number (index of an image. Not the index of the array, but incremented by 1)
firstImage

An ImageEvent right after that the FIRST image in images is visible.

It contains an Action enum and a result (boolean or number)

  • action: either Action.NORMAL, Action.CLICK, Action.KEYBOARD or Action.SWIPE
  • result: true
lastImage

An ImageEvent right after that the LAST image in images is visible.

It contains an Action enum and a result (boolean or number)

  • action: either Action.NORMAL, Action.CLICK, Action.KEYBOARD or Action.SWIPE
  • result: true


CarouselConfig

Interface to configure the carousel, like to change sizes, hide some features and so on.


maxWidth
Type: string

Maximum width of the carousel. It's used to set the CSS property maxWidth. '100%' (full width) by default.
maxHeight
Type: string

Maximum height of the carousel. It's used to set the CSS property maxHeight. '400px%' by default.
showArrows
Type: boolean

boolean to disable both left and right arrows. true by default.
objectFit
Type: string

CSS objectFit property. For more information check Mozilla MDN web doc. 'cover' by default.
keyboardEnable
Type: boolean

boolean to disable keyboard navigation. true by default.
modalGalleryEnable
Type: boolean

boolean to enable clicks on the current image to open modal gallery. false by default.
legacyIE11Mode
Type: boolean

boolean to enable legacy support for Internet Explorer 11 (IE11), otherwise the carousel won't work properly. false by default.

CarouselImageConfig

Interface to change current image of the carousel, like description or to invert the swipe direction.


description
Type: stringoptional

Optional description text. Attention: this is not the full description, but only the last part, after image numbers. For instance: `Image 2/12 - description`. To ignore this feature, don't add this property.
invertSwipe
Type: booleanoptional

Optional boolean property to invert the default swipe direction on devices with touchscreen.

ImageEvent

The payload of events related to action, emitted by this library.


galleryId
Type: number

Unique id of the current carousel.
action
Type: Action

It's the enum Action
result
Type: number | boolean

The result/data of the emitted event. For example, it could be the index of the current image or a boolean to say if images are ready to display and so on.
