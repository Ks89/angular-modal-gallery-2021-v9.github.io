Modal Gallery

Modal Gallery is the main part of @ks89/angular-modal-gallery. It shows the navigable collection of Images with buttons, previews and many other customizable features to fit your needs


Live demo



API

Follow the getting started guide to install @ks89/angular-modal-gallery and all requirements.

To use Modal Gallery in your application, you have to invoke the .open(...) method of ModalGalleryService. You are free to invoke this method wherever you want, for instance pressing a button.

Previous versions of @ks89/angular-modal-gallery were very limited forcing you to add a selector into the HTML file, however, now you can show Modal Gallery without adding UI element to you app, or even automatically after an event, invoking the open method.

Inputs

id
Type: number
A mandatory unique id >= 0 of the current modal gallery, used by GalleryService.
images
Type: Image[]
The mandatory array of Image objects, that represent the modal images. In fact, an Image is an object with image source paths and other additional configuration parameters. For more info check Image. Please don't use this.images.push(img), when you want to modify this array, instead re-assign it to trigger change detection. For instance: this.images = [...this.images, newImage] to create a new array from the original with a new image at the end.
Attention: if you need to add the same image (same url) multiple times in the same modal gallery, you must append to the url a random but unique query param to prevent caching issues. For more info, please read this issue.
currentImage
Type: Image
The image object to show as soon the modal gallery opens. You should simply pass one of the element of the images array. For instance, if you want to open the first image in the array, pass images[0].
config
Type: LibConfigoptional

Interface to configure modal gallery.

Outputs

close

An ImageModalEvent right before that the modal view closes.

It contains an Action enum and a result (boolean or number)

  • action: Action that can be NORMAL, KEYBOARD or CLICK
  • result: true
show

An ImageModalEvent right before that the modal image will be visible.

It contains an Action enum and a result (boolean or number).

Everytime the current image changes, a new ImageModalEvent will be emitted.

  • action: either Action.NORMAL, Action.CLICK, Action.KEYBOARD, Action.SWIPE, Action.AUTOPLAY or Action.SWIPE
  • result: number (index of an image. Not the index of the array, but incremented by 1)
firstImage

An ImageModalEvent right before that the FIRST image in modalImages is visible.

It contains an Action enum and a result (boolean or number)

  • action: either Action.NORMAL, Action.CLICK, Action.KEYBOARD, Action.SWIPE, Action.AUTOPLAY or Action.SWIPE
  • result: true
lastImage

An ImageModalEvent right before that the LAST image in modalImages is visible.

It contains an Action enum and a result (boolean or number)

  • action: either Action.NORMAL, Action.CLICK, Action.KEYBOARD, Action.SWIPE, Action.AUTOPLAY or Action.SWIPE
  • result: true
hasData

An ImageModalEvent right after the initialization of modalImages.

It contains an Action enum and a result (boolean or number)

This event is useful to know when angular-modal-gallery is ready.

  • action: Action.LOAD
  • result: true
buttonBeforeHook

A ButtonEvent right before the action related to an upper button.

It contains an Action enum, the current Image and the ButtonConfig that triggered this event

This event is a 'before hook', because you can execute your code before every action triggered by buttons. Also, since you have the full ButtonConfig in the payload, you can run different code based on button's type

  • action: Action.LOAD
  • button: ButtonConfig
  • image: Image
buttonAfterHook

A ButtonEvent right after the action related to an upper button.

It contains an Action enum, the current Image and the ButtonConfig that triggered this event

This event is an 'after hook', because you can execute your code after every action triggered by buttons. Also, since you have the full ButtonConfig in the payload, you can run different code based on button's type

  • action: Action.LOAD
  • button: ButtonConfig
  • image: Image


ModalImage

Interface that represent the modal image.


img
Type: string | SafeResourceUrl

Mandatory source path of the modal image. To support base64 images, check the official example.

If you need multiple non-base64 images with the same image path, to prevent caching issues you have to append something like '?index', for instance '?0', '?1'. For more information, please check the official example called 'Same images'.
description
Type: stringoptional

Optional description text. Attention: this is not the full description, but only the last part, after image numbers. For instance: `Image 2/12 - description`. To ignore this feature, don't add this property.
extUrl
Type: stringoptional

Optional external URL path. You must use a valid URL, because angular-modal-gallery will simply navigate to this URL. To ignore this feature, don't add this property.
title
Type: stringoptional

Optional title text. It sets 'title' html's attribute, useful for accessibility purposes. To ignore this feature, don't add this property.
alt
Type: stringoptional

Optional alt text. It sets 'alt' html's attribute, useful for accessibility purposes. To ignore this feature, don't add this property.
arialLabel
Type: stringoptional

Optional aria label text. It sets 'aria-label' html's attribute, useful for accessibility purposes. To ignore this feature, don't add this property.
invertSwipe
Type: booleanoptional

Optional boolean property to invert the default swipe direction on devices with touchscreen.

ImageModalEvent

The payload of events related to action, emitted by this library.


action
Type: Action

It's the enum Action
result
Type: number | boolean

The result/data of the emitted event. For example, it could be the index of the current image or a boolean to say if images are ready to display and so on.

CurrentImageConfig

Interface to customize current image in modal gallery.


downloadable
Type: booleanoptional

If true, you'll be able to download the current modal image always with keyboard shortcut (ctrl+s or cmd+s), but the download button will be visible only when showDownloadButton=true. Otherwise, all download features will be disabled. downloadable=false by default

Please be careful: on SmartPhones and Tablets you can't use keyboard shortcuts, so to download images, please enable downloadable and use ButtonsStrategy.SIMPLE or higher to add the download button.
loadingConfig
Type: LoadingConfigoptional

Interface to hide/show and to configure the modal loading spinner. { enable: true, type: 1 } by default.
description
Type: Description interfaceoptional

Interface to specify a custom description either a full string or an object with string properties. The first one is the entire string (for instance 'Description of Image 1/2 - My family - bla bla'), the second one is an object with these properties imageText (default is 'Image'), numberSeparator (default is '/'), beforeTextDescription (default is ' - '). You can also use HTML descriptions, but remember that Angular will sanitize dangerous code, so for instance you cannot add input fields and so on, but only simple and secure HTML.

To customize part of the description, you can refer to this example: `imageText 2 numberSeparator 12 beforeTextDescription bla bla`. As you can see, "red" parts are customizable using the three properties described above. Furthermore, `2` is the index of the visible image, `12` the total number of images and `bla bla` the description passed thanks to the input Image object.

Attention please! There aren't spaces between values. So if you want a 'text - text' you have to specify exactly ' - ', with the two spaces

Also, from version 5.4.0, you can use this interface to change description's style.
navigateOnClick
Type: booleanoptional

If false, click events on the current image will be ignored, otherwise will navigate to the next image.
navigateOnClick
Type: booleanoptional

If false, click events on the current image will be ignored, otherwise will navigate to the next image.

GalleryService

Service to interact with modal gallery without clicking on plain-gallery. For instance, you can open modal gallery from your Component's code instead of clicking on a thumb in plain gallery.


openGallery ( galleryId, index )
Input types: galleryId: number | undefined  and  index: number

Method to open an image (index) of modal gallery (id). You have to provide both an index >= 0 and a galleryId >= 0. Also, remember that ids must be unique across your web page, so every instance of modal gallery must have a different id.
closeGallery ( galleryId )
Input types: galleryId: number | undefined

Method to close the modal gallery with id equals to the input parameter galleryId. You have to provide a galleryId >= 0. Also, remember that ids must be unique across your web page, so every instance of modal gallery must have a different id.
update ( galleryId, index, newImage )
Input types: galleryId: number | undefined  ,   index: number and  newImage: Image

Method to update the images array at position index with newImage passed as parameter. You have to provide a galleryId >= 0. Also, remember that ids must be unique across your web page, so every instance of modal gallery must have a different id.

LoadingType

Enum that represents the type of the loading spinner. It can be one of these enum values:


  • STANDARD: the default spinner
  • CIRCULAR: a circular spinner
  • BARS: spinner built with some animated vertical bars
  • DOTS: spinner composed by animated dots
  • CUBE_FLIPPING: flipping cube spinner
  • CIRCLES: spinner composed by multiple circles
  • EXPLOSING_SQUARES: spinner composed by 4 squares moving outside

KeyboardEnum

Enum that represents keyboard's keycodes. It can be one of these enum values:


  • ESC: 27
  • LEFT_ARROW: 37
  • RIGHT_ARROW: 39
  • UP_ARROW: 38
  • DOWN_ARROW: 40
